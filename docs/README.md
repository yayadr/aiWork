# docs/

存放项目的正式文档。

## 用途

- 项目架构说明
- API 接口文档
- 部署运维指南
- 需求设计文档

## 收录规则

- 文档应描述稳定的项目知识，并注明适用范围和更新时间
- 临时协作过程放入 `sessions/`，可复用提示词放入 `prompts/`
- 文档不得包含真实凭据、环境变量值或其他敏感信息

## 结构

按项目模块划分子目录，如：

```
docs/
├── architecture/    # 架构设计
├── api/             # 接口文档
├── deploy/          # 部署文档
└── changelog/       # 变更日志
```

仅在确有对应文档时创建子目录，避免保留空目录结构。
